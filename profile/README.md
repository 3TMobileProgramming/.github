## 🌟기획 배경🌟
기존의 학교 공지사항은 게시판 형태로 제공되어 원하는 정보를 찾기 위해 여러 게시글을 직접 확인해야 하는 불편함이 있다. 본 시스템은 자연어 질문을 이해하는 AI 기술을 활용하여 사용자가 궁금한 내용을 질문하면 관련된 공지사항을 검색·분석하여 필요한 정보를 빠르게 제공한다.<br><br>

이를 통해 학생들은 복잡한 공지 게시판을 직접 탐색하지 않고도 필요한 정보를 효율적으로 얻을 수 있으며, 학교 공지 접근성을 크게 향상시킨다.<br>
또한 챗봇 형식을 활용해 자연어로 묻고, 자연어로 대답을 받는 문답 형식의 공지 검색 알고리즘을 통해, 사용자의 편의성과 접근성을 향상시킬 수 있다.<br>

## 🌟3TEAM 소개🌟
### 정세영- PM 

- notion 기반 팀프로젝트 진행

- 간트 차트, 주간 업무 보고서로 일정 관리
- github로 코드 취합, 단위/통합 테스트 진행
  

### 이정우- 프론트엔드(화면 설계, 구현)

- 개발 환경: Android Studio (Android Native 환경)
- 개발 언어: Java, XML
- 네트워크 통신: Retrofit2 (서버 데이터 송수신용)
- 이미지/애니메이션: Glide (이미지 처리, 로딩 애니메이션)
- UI 컴포넌트: Material Design, ConstraintLayout, Markwon
  

### 김수현- 백엔드(크롤링, DB)
- 크롤러: Jsoup 기반 HTML 파싱 및 URL 필터링
- 검색: MySQL FULLTEXT + N-gram 파서 적용 한국어 검색 고도화
- Spring Boot + JPA / JDBC 혼용
- 데이터베이스: MySQL
- 공지 조회/검색 REST API 구현 (전체/카테고리/키워드)
- @Scheduled 자동 크롤링 스케줄러 구현 (매일 오전 8시)
- ON DUPLICATE KEY UPDATE 중복 데이터 방지 처리
- 프론트팀 API 명세서 작성 및 연동 지원

  
### 박성현- 백엔드(DB 키워드 추출, gpt api 연결)
- Spring Boot + JPA
- 데이터베이스: MySQL
- AI Response Token Model : ChatGPT

### 🔥공통<br>
#### 👉기술성<br>
1. 학교 공지사항 데이터를 수집하고 검색 가능한 데이터 구조로 구축한다.<br>
2. 사용자의 자연어 질문을 분석하여 관련 공지사항을 찾아주는 AI 기반 검색 기능을 구현한다.<br>
3. 사용자 친화적인 인터페이스를 통해 누구나 쉽게 공지를 검색할 수 있는 애플리케이션을 구현한다.<br>
4. 질문과 관련된 공지 내용을 요약하거나 핵심 정보를 제공하는 지능형 응답 시스템을 개발한다.<br>
5. 모바일 어플리케이션 파일을 구축하여 다운받을 수 있는 형태로 제작한다.<br><br>

이를 통해 공지사항 검색의 편의성과 정보 접근성을 향상시키는 것을 목표로 한다.<br><br><br>


#### 👉제작과정<br>
애자일 개발론을 활용, 간트 차트와 주간업무보고서를 작성하여 과제 진행 상황을 공유하고, 매주 테스트를 진행하며 프로젝트를 수행한다.<br>
- **데이터 수집 및 전처리** <br>
학교 홈페이지 또는 공지 게시판에서 공지 데이터를 수집<br>
공지 제목, 내용, 날짜, 카테고리 등의 정보를 정리하여 데이터베이스에 저장<br>
- **검색 시스템 구축**<br>
키워드 기반 검색 및 자연어 기반 검색 기능 구현<br>
질문과 관련된 공지사항을 효율적으로 찾을 수 있도록 검색 알고리즘 설계<br>
- **AI 질문 응답 기능 구현**<br>
사용자의 질문을 분석하여 관련 공지를 찾고 핵심 내용을 제공<br>
필요 시 공지 내용을 AI를 이용해 자연어 형식으로 요약하여 사용자에게 전달<br>
- **애플리케이션 개발**<br>
웹 또는 모바일 환경에서 사용할 수 있는 사용자 인터페이스 설계<br>
검색 결과를 직관적으로 보여주는 UI 구현<br>
- **테스트 및 개선**<br>
실제 사용자 시나리오를 기반으로 기능 테스트 수행<br>
검색 정확도와 응답 품질을 개선<br><br><br>

### 🔥프론트엔드
- **앱 실행 및 검색**
    - 앱 실행 시 메인 화면 중앙에 직관적인 검색창을 배치하여 사용자가 구어체 질문을 입력할 수 있도록 구성
    - '장학금', '수강신청' 등 자주 찾는 키워드는 칩(Chip) 버튼으로 제공하여 원터치 검색 지원
- **로딩 및 상태 표시**
    -  AI 분석 시간 동안 Glide 애니메이션 등을 활용해 시각적 피드백 제공
- **결과 출력**
    - 상단 (AI 요약): AI가 분석해서 보내준 답변 내용을 Markwon 라이브러리를 통해 가독성 좋게 배치
    - 하단 (출처 원문): 답변의 출처인 실제 학교 공지사항을 나열
- **상세 원문 확인**
    - 하단의 공지를 클릭하면 외부 브라우저를 통해 학교 홈페이지의 해당 공지 글로 즉시 연결
- **예외 상황 처리**:
    - 검색 결과가 없거나 네트워크 오류 시 안내 문구와 일러스트 노출
    - 통신 실패 시 이전 검색어를 기억하여 재요청하는 '다시 시도' 로직 구현

**<작동 원리>**
1. **질문 전송**: 사용자가 검색어 입력, 전송 버튼 클릭 시 화면을 로딩 상태(Glide)로 전환, Retrofit2를 통해 검색어 데이터를 서버로 전송
2. **비동기 처리**: 서버 분석 대기 시간 동안 UI 스레드가 멈추지 않도록 처리
3. **데이터 바인딩**: 서버 응답 성공 시, 상단 텍스트뷰에는 요약본을, 하단 공지 리스트를 갱신하며, 실패 시 에러 화면과 '다시 시도' 기능을 제공


### 🔥 백엔드

#### 👉 Flow
```
SyuNoticeCrawler (URL 수집)
        ↓
SyuNoticeParser (내용 추출)
        ↓
CrawledNotice (데이터 담기)
        ↓
JdbcNoticeRepository (DB 저장)
        ↓
NoticeRepository (DB 조회)
```

#### 👉 데이터 수집 방식
- 학사/장학/행사 카테고리별 각 100건씩 수집
- 매일 오전 8시 @Scheduled 자동 크롤링
- ON DUPLICATE KEY UPDATE로 중복 저장 방지
- URL 기준 UNIQUE KEY 설정으로 데이터 무결성 유지

#### 👉 질문 처리 흐름
```
사용자 질문 입력
        ↓
앱에서 서버로 요청
        ↓
키워드 추출 및 정규화
        ↓
DB 검색 (FULLTEXT + N-gram)
        ↓
GPT-4o-mini 전달
        ↓
답변 생성 후 결과 반환
```

#### 👉 주요 기능
- **공지사항 수집**
    - 공지 URL 기반 수집 및 본문 추출 후 DB 저장
    - ON DUPLICATE KEY UPDATE로 중복 저장 방지
    - 매일 오전 8시 자동 크롤링 스케줄러 실행
- **검색 기능**
    - MySQL FULLTEXT + N-gram 파서 적용으로 한국어 검색 정확도 개선
    - 카테고리별 필터링 검색 지원
    - 키워드 기반 공지 검색 API 제공
- **사용자 질문 처리**
    - 자연어 질문 → 키워드 정규화 → DB 검색 → GPT 전달 → 답변 생성
- **예외 처리**
    - 공지 없음 / GPT 실패 / 네트워크 오류 대응

#### 👉 크롤링 파트
| 파일 | 역할 |
|------|------|
| SyuNoticeCrawler.java | 학교 공지사항 목록 페이지에서 URL 수집. Jsoup으로 HTML 읽고 불필요한 링크 필터링 |
| SyuNoticeParser.java | 수집된 URL에 접속해서 제목/날짜/본문 추출. 불필요한 태그 제거 후 정제 |
| CrawledNotice.java | 크롤링/파싱 결과를 담는 객체. 제목/날짜/본문/URL/카테고리를 하나로 묶어 저장 |
| NoticeCrawlerService.java | 크롤러/파서/DB저장을 묶어 자동 실행. @Scheduled로 매일 오전 8시 순차 실행 |

#### 👉 DB 파트
| 파일 | 역할 |
|------|------|
| JdbcNoticeRepository.java | 파싱된 데이터를 MySQL에 저장. ON DUPLICATE KEY UPDATE로 중복 방지. MySQL 전용 문법 때문에 JDBC 직접 사용 |
| NoticeRepository.java | JPA 인터페이스로 DB 조회. 메서드 이름만으로 쿼리 자동 생성. 전체/카테고리별/키워드 검색 지원 |
| Notice.java | notices 테이블과 1대1 매핑. id/title/date/url/body/category/fetchedAt 컬럼 정의 |
| application.properties | DB 주소, 계정, 포트 등 연결 설정 파일 |
